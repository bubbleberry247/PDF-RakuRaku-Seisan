# Scenario 55 業務ルール（必読 / セッション開始時に読むこと）

## 1. 支払スケジュール
- 支払期限: **15日 / 25日 / 末日**（月3回）
- 15日=固定支払い（家賃,リース）、25日=グループ会社間、末日=一般取引先
- 作業開始: 支払日の**約1週間前～4営業日前**（例: 15日払い→9日頃）
- 銀行アップロード期限: 支払日の**土日祝除く3営業日前**
- 締め後の申請 → 都度払い（吉田さんがインターネットバンキングで個別処理）
- **実行方法: 手動GUI（タスクスケジューラ不要）**

## 2. データ抽出条件
- 対象: **一般経費のみ**（現場経費は対象外）
- 支払方法: **総合振込のみ**（都度振込・口座振替は除外）
- 承認ステータス: 確認済 / 確認 / 確定待（承認依頼中は除外）
- 検索キー: 楽楽精算の**支払予定日**（年月日指定検索）
- 会計区分振り分け: **廃止**（すべて未払費用セクションに統一転記）
- 振込手数料: **すべて当方負担**（判定不要）

## 3. Excel転記ルール
- ブロック構造: 1シートに15日/25日/末日エリア（銀行照合のため合計行必須）
- 書き込み可能列: **C列（摘要）とI列（支払額）のみ**
- 書き込み禁止: H列（税計算数式）、J列（合計数式）、偶数行（税行）
- テンプレートマスタ列: A（業者番号）, E（科目）, F（科目コード）
- D列（店コード）, G列（税区分）: テンプレート固定値不可（動的取得要）
- 同一支払先の複数明細: 結合せず個別行で転記
- 按分: RPAは1行転記、吉田さんが手動分割

## 4. 未登録支払先・エラー処理
- 未登録支払先: A列=**6799**、赤字で区別、吉田さん確認
- 金額未取得時: I列にダミー値**1**を仮入力、赤字表示
- 空行不足（オーバーフロー）: 処理可分だけ転記、未転記分をレポート通知
- スクレイピング取りこぼし: アイコン文字（📎等）除去必要

## 5. 科目補正（優先順位）
1. 補正テーブル（既知の誤り→正解マッピング）
2. 過去最頻出科目（信頼度100%）
3. 類似支払先推定
4. キーワード辞書
5. デフォルト: 消耗品費（8440）
