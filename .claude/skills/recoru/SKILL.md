# レコル総合スキル

レコル（RecoRu）勤怠管理システムに関する質問に、ドメイン知識ドキュメントを参照して回答するスキル。

## このスキルでできること

1. **勤務区分の判定** - 有休/振休/代休/公休等の違いと使い分け
2. **申請手順の案内** - 各種申請の入力項目と注意点
3. **帳票の見方** - 勤怠一覧表、残業一覧表等の確認ポイント
4. **例外ケース対応** - 締め後修正、承認者不在、打刻漏れ等の対処法

質問に応じて以下のドキュメントを参照する：

## 回答時の指示

### 1. 必ずドキュメントを参照する

回答前に必ず関連ドキュメントを読み、**ドキュメントに書かれた内容に基づいて**回答する。
推測や一般論ではなく、プロジェクト固有のルールを優先する。

### 2. 勤務区分の質問には辞書を使う

「有休と振休の違いは？」「代休とは？」などの質問には、
必ず `10_kbn_dictionary.md` を読んでから回答する。

### 3. 例外ケースの質問には例外集を使う

「締め後に修正したい」「承認者がいない」などの質問には、
必ず `40_edge_cases.md` を読んでから回答する。

### 4. TBD項目がある場合は正直に伝える

ドキュメントにTBDと書かれている項目は「未確定」と伝え、
担当者への確認を促す。

### 5. RPA対応可否を明示する

例外ケースについては、RPAで自動化可能か、手動対応が必要かを明示する。

## 使用例

### 例1: 勤務区分の質問

**ユーザー**: 振休と代休の違いは？

**回答手順**:
1. `10_kbn_dictionary.md` を読む
2. 「振休と代休の違い」セクションを参照
3. 表形式で違いを説明

### 例2: 例外ケースの質問

**ユーザー**: 締め後に勤怠を修正したい

**回答手順**:
1. `40_edge_cases.md` を読む
2. 「EC-締01: 締め後の勤怠修正」を参照
3. 対処手順、必要権限、RPA対応可否を説明

### 例3: 申請手順の質問

**ユーザー**: 有休申請の手順は？

**回答手順**:
1. `30_application_howto.md` を読む
2. 「2.1 有給休暇申請」を参照
3. 画面遷移と入力項目を説明
