## 1. UI変更

### 特徴的なエラー

- `TimeoutError: Timeout 30000ms exceeded`
- `Error: Element not found: button[type="submit"]`
- `Selector resolved to empty`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **screenshots/** | エラー時の画面キャプチャ。UI要素の有無、位置、デザイン変更を確認 |
| **trace.zip** | Playwright Trace Viewerで操作履歴を確認。どのセレクタで止まったか |
| **execution.log** | `Timeout waiting for selector` 等のログを検索 |

### チェックリスト

- [ ] セレクタが無効になっていないか？（id/class名の変更）
- [ ] ボタン/リンクが削除されていないか？
- [ ] 動的要素の読込タイミングが変わっていないか？
- [ ] モーダル/ポップアップが追加されていないか？
- [ ] アプリのバージョンアップがなかったか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| id/class名変更 | `data-testid` に変更（UI変更に強い） |
| 動的ID | `role` / `label` に変更 |
| モーダル追加 | モーダル検出 → 閉じる処理を追加 |
| 読込遅延 | `waitForSelector` / `waitForLoadState` 追加 |

### 再実行判定

❌ **再実行不可**（コード修正必要）

## 2. 認証エラー

### 特徴的なエラー

- HTTP 401 Unauthorized
- HTTP 403 Forbidden
- `Login failed` （ログインループ）
- `Invalid credentials`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **network/trace.har** | API呼び出しのレスポンスコード（401/403） |
| **screenshots/** | ログイン画面がループしていないか |
| **execution.log** | `Authentication failed` 等のログ |
| **metadata.json** | 実行ユーザー名の確認 |

### チェックリスト

- [ ] 資格情報（ID/パスワード）が正しいか？
- [ ] パスワード有効期限が切れていないか？
- [ ] アカウントがロック/無効化されていないか？
- [ ] 2要素認証が追加されていないか？
- [ ] API tokenが期限切れになっていないか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| パスワード期限切れ | 定期的なパスワード更新フロー |
| アカウント無効化 | 実行前のアカウント状態確認 |
| 2要素認証追加 | APIキー認証への切り替え |
| Token期限切れ | Token自動更新ロジック |

### 再実行判定

❌ **再実行不可**（資格情報/設定の修正必要）

## 3. 権限エラー

### 特徴的なエラー

- HTTP 403 Forbidden
- `EACCES: permission denied`
- `Access denied`
- `Cannot write to directory`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **execution.log** | `EACCES` / `permission denied` 等のログ |
| **metadata.json** | 実行ユーザー、マシン名 |
| **network/trace.har** | API呼び出しの403レスポンス |

### チェックリスト

- [ ] ファイル/フォルダの書込権限があるか？
- [ ] ネットワーク共有フォルダへのアクセス権があるか？
- [ ] APIアクセス権限が付与されているか？
- [ ] 実行ユーザーが変わっていないか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| フォルダ権限不足 | 実行ユーザーに書込権限付与 |
| 共有フォルダアクセス拒否 | ネットワーク認証情報の更新 |
| API権限不足 | 管理者に権限追加を依頼 |

### 再実行判定

❌ **再実行不可**（権限設定の修正必要）

## 4. ネットワーク障害

### 特徴的なエラー

- `ETIMEDOUT`
- `ECONNREFUSED`
- `ECONNRESET`
- `getaddrinfo ENOTFOUND` （DNS解決失敗）

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **network/trace.har** | 失敗したリクエストのURL、タイムアウト |
| **execution.log** | `ETIMEDOUT` 等のログ |
| **metadata.json** | 実行時刻（業務ピーク時間帯か？） |

### チェックリスト

- [ ] インターネット接続が切断されていないか？
- [ ] 対象サーバーがダウンしていないか？
- [ ] 社内Proxyが応答しているか？
- [ ] ファイアウォールでブロックされていないか？
- [ ] 業務ピーク時間帯（12:00-13:00等）ではないか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| 一時的なネットワーク遅延 | リトライロジック追加（exponential backoff） |
| サーバーダウン | 事前ヘルスチェック、監視アラート |
| Proxy応答遅延 | タイムアウト延長、Proxy設定見直し |
| ピーク時間帯の遅延 | 実行時間帯の調整 |

### 再実行判定

✅ **再実行可**（一時的障害の場合のみ）
- 再実行前に対象サーバーへのpingを確認
- 業務時間外に再実行

## 5. Proxy/SSL証明書

### 特徴的なエラー

- `UNABLE_TO_VERIFY_LEAF_SIGNATURE`
- `CERT_HAS_EXPIRED`
- `ERR_SSL_PROTOCOL_ERROR`
- `Proxy connection failed`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **network/trace.har** | SSL/TLS handshake失敗、証明書エラー |
| **execution.log** | `SSL` / `certificate` 等のログ |
| **metadata.json** | 実行環境（Proxy設定が異なるか？） |

### チェックリスト

- [ ] Proxy設定が正しいか？（HTTP_PROXY, HTTPS_PROXY）
- [ ] SSL証明書が期限切れでないか？
- [ ] 自己署名証明書を使用していないか？
- [ ] 社内Proxyのルート証明書がインストールされているか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| Proxy設定不足 | 環境変数でProxy設定 |
| SSL証明書期限切れ | 証明書更新、期限監視 |
| 自己署名証明書 | `rejectUnauthorized: false`（開発環境のみ） |
| Proxyルート証明書未インストール | 証明書インストール手順書作成 |

### 再実行判定

❌ **再実行不可**（Proxy/証明書設定の修正必要）

## 6. タイムアウト

### 特徴的なエラー

- `TimeoutError: Timeout 30000ms exceeded`
- `Navigation timeout of 30000 ms exceeded`
- `Waiting for selector "..." failed: timeout`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **execution.log** | タイムアウト発生箇所、待機時間 |
| **trace.zip** | 何を待っていたか（selector, navigation, response） |
| **network/trace.har** | API呼び出しの応答時間 |

### チェックリスト

- [ ] タイムアウト時間は適切か？（SaaSは遅いので60秒推奨）
- [ ] 待機条件が正しいか？（selector, URL, response）
- [ ] ネットワーク遅延が発生していないか？
- [ ] サーバー側の処理が遅延していないか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| タイムアウト時間不足 | 30秒 → 60秒 に延長 |
| 待機条件が不適切 | `waitForSelector` → `waitForURL` に変更 |
| サーバー処理遅延 | リトライロジック、実行時間帯調整 |
| ネットワーク遅延 | `waitForLoadState('networkidle')` 追加 |

### 再実行判定

△ **条件付き再実行可**
- 一時的な遅延の場合は再実行可
- 恒常的なタイムアウトの場合はコード修正必要

## 7. データ不整合

### 特徴的なエラー

- `Cannot read property 'xxx' of null`
- `Unexpected value: null`
- `Array is empty`
- `Validation failed: amount must be positive`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **input/** | 入力データの内容（null, 空配列, 想定外の値） |
| **execution.log** | データ検証エラーのログ |
| **screenshots/** | フォーム入力時のエラーメッセージ |

### チェックリスト

- [ ] 入力データが想定形式か？（JSON schema検証）
- [ ] 必須項目が欠けていないか？
- [ ] 数値範囲が正しいか？（負の値、0、上限超過）
- [ ] 日付形式が正しいか？（YYYY-MM-DD等）
- [ ] 文字列長が制限内か？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| 入力データ検証不足 | JSON schema検証を追加 |
| 上流システムのデータ不正 | 上流に修正依頼、暫定的にフィルタリング |
| 型変換エラー | 明示的な型変換とエラーハンドリング |

### 再実行判定

❌ **再実行不可**（入力データの修正必要）

## 8. リソース不足

### 特徴的なエラー

- `ENOMEM: Cannot allocate memory`
- `ENOSPC: No space left on device`
- `Out of memory`
- `Maximum call stack size exceeded`

### 確認すべき証跡

| 証跡 | 確認内容 |
|------|---------|
| **metadata.json** | メモリ、ディスク容量 |
| **execution.log** | `ENOMEM` / `ENOSPC` 等のログ |

### チェックリスト

- [ ] ディスク空き容量は十分か？（最低10GB）
- [ ] メモリ使用量は適切か？（大量データ読込時）
- [ ] ブラウザコンテキストをリークしていないか？
- [ ] 一時ファイルが溜まっていないか？

### 典型的な根本原因

| 原因 | 対策 |
|------|------|
| ディスク容量不足 | 古いartifactの自動削除 |
| メモリリーク | ブラウザコンテキストの適切なclose |
| 大量データ読込 | ストリーム処理、バッチ分割 |
| 一時ファイル未削除 | クリーンアップスクリプト |

### 再実行判定

△ **条件付き再実行可**
- リソース解放後は再実行可
- 恒常的な不足の場合はリソース増強が必要

## 調査手順（推奨フロー）

```
1. metadata.json読み取り
   ↓
2. エラー型で原因カテゴリを絞り込み
   ↓
3. 該当カテゴリのチェックリストを実行
   ↓
4. 証跡で根本原因を特定
   ↓
5. 再実行判定
   ↓
6. Incident Report作成
```

## クイックリファレンス表

| エラーメッセージ | カテゴリ | 再実行可否 |
|----------------|---------|-----------|
| TimeoutError | UI変更 or タイムアウト | ❌ / △ |
| 401/403 | 認証 or 権限 | ❌ |
| ETIMEDOUT | ネットワーク | ✅ |
| CERT_INVALID | Proxy/SSL | ❌ |
| Cannot read property 'xxx' of null | データ不整合 | ❌ |
| ENOSPC | リソース不足 | △ |

**凡例**:
- ✅ 再実行可
- ❌ 再実行不可（修正必要）
- △ 条件付き再実行可
