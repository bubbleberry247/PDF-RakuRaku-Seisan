# 期待される成果物ツリー例

## 成功実行の例（最小セット）

artifacts/
└── rakuraku-expense-approval/
    ├── 20260214-153042-abc123/              # runId
    │   ├── metadata.json                    # status: "success"
    │   ├── execution.log                    # ログ（INFO/WARNのみ）
    │   ├── input/
    │   │   ├── hash.txt                     # SHA-256: abc123...
    │   │   └── sample.json                  # 入力データサンプル（個人情報マスク済み）
    │   └── output/
    │       └── expense-report.xlsx          # 出力ファイル
    ├── latest -> 20260214-153042-abc123     # シンボリックリンク（最新）
    └── last-success -> 20260214-153042-abc123  # シンボリックリンク（最終成功）

サイズ: 約2MB（ログ1MB + 入力サンプル500KB + 出力500KB）

## 失敗実行の例（完全セット）

artifacts/
└── rakuraku-expense-approval/
    ├── 20260214-160500-def456/              # runId（失敗）
    │   ├── metadata.json                    # status: "failed", error: {...}
    │   ├── execution.log                    # ログ（ERRORあり）
    │   ├── trace.zip                        # Playwright trace（15MB）
    │   ├── screenshots/
    │   │   ├── error-001.png                # エラー画面1
    │   │   └── error-002.png                # エラー画面2
    │   ├── network/
    │   │   └── trace.har                    # ネットワークログ（5MB）
    │   ├── input/
    │   │   ├── hash.txt
    │   │   └── sample.json
    │   └── dump/
    │       └── error-page.html              # エラー時のHTML全体
    ├── 20260214-153042-abc123/              # runId（過去の成功）
    │   └── ...
    ├── latest -> 20260214-160500-def456     # シンボリックリンク（最新=失敗）
    └── last-success -> 20260214-153042-abc123  # シンボリックリンク（最終成功）

サイズ: 約25MB（trace 15MB + HAR 5MB + screenshots 1MB + その他4MB）

## 注意事項

1. **シンボリックリンク**:
   - Windowsでは管理者権限が必要な場合あり
   - 代替: `latest.txt` に最新runIdを記録

2. **サイズ制限**:
   - 1つのrunIdが100MBを超えたら警告
   - videos/は500MB上限（デバッグ時のみ）

3. **保持期間**:
   - 成功: 7日
   - 失敗: 30日
   - デバッグ: 90日
   - latest/last-successは無期限

4. **アクセス権限**:
   - artifacts/は管理者のみ
   - 共有時はzipにパスワード設定
