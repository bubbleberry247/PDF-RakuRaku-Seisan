## 2. 各申請の詳細手順

### 2.1 有給休暇申請

#### 概要
- **目的**: 年次有給休暇を取得する
- **事前条件**: 有休残日数が1日以上
- **承認フロー**: 申請者 → 直属上長 → (TBD)

#### 画面遷移

```
1. レコルログイン
   ↓
2. メニュー → 申請 → 休暇申請
   ↓
3. 申請種別選択 → 有給休暇
   ↓
4. 入力画面
   ↓
5. 確認画面
   ↓
6. 完了画面
```

#### 入力項目

| 項目名 | 必須 | 入力形式 | 説明 | 注意点 |
|--------|------|---------|------|--------|
| 取得日 | ○ | 日付 | 休暇を取得する日 | 過去日は不可（TBD） |
| 取得日（終了） | △ | 日付 | 複数日の場合の終了日 | 連続取得時のみ |
| 理由 | TBD | テキスト | 取得理由 | 必須かどうか要確認 |
| TBD | TBD | TBD | TBD | TBD |

#### 注意点・よくある質問

| 質問・状況 | 回答・対処 |
|-----------|-----------|
| 残日数が足りない | 申請不可。特別休暇や欠勤を検討 |
| 過去日に申請したい | TBD（打刻修正？） |
| 承認者が不在 | [40_edge_cases.md](./40_edge_cases.md) 参照 |

#### 画面スクリーンショット
- 申請画面: `docs/assets/recoru/screenshots/TBD.png`

### 2.2 半休申請

#### 概要
- **目的**: 半日の有給休暇を取得する
- **事前条件**: 有休残日数が0.5日以上
- **承認フロー**: 申請者 → 直属上長

#### 入力項目

| 項目名 | 必須 | 入力形式 | 説明 | 注意点 |
|--------|------|---------|------|--------|
| 取得日 | ○ | 日付 | 休暇を取得する日 | - |
| 午前/午後 | ○ | 選択 | 午前半休 or 午後半休 | - |
| TBD | TBD | TBD | TBD | TBD |

#### 注意点

| 状況 | 対処 |
|------|------|
| 半休と全休を同日申請 | 不可。全休のみ申請 |
| 午前半休後に残業 | 午後の勤務時間に応じて計算 |

### 2.3 振替休日申請

#### 概要
- **目的**: 休日出勤の代わりに別日を休む
- **事前条件**: 休日出勤の予定または実績がある
- **承認フロー**: 申請者 → 直属上長

#### 入力項目

| 項目名 | 必須 | 入力形式 | 説明 | 注意点 |
|--------|------|---------|------|--------|
| 振替元日（休日出勤日） | ○ | 日付 | 出勤する休日 | - |
| 振替先日（休暇日） | ○ | 日付 | 代わりに休む日 | 同一週内推奨 |
| TBD | TBD | TBD | TBD | TBD |

#### 振休と代休の選択基準

```
休日出勤の予定がある
├─ 出勤前に申請する → 振替休日申請
└─ 出勤後に申請する → 代休申請
```

詳細は [10_kbn_dictionary.md](./10_kbn_dictionary.md) の「振休と代休の違い」参照

## 3. 申請後のフロー

### 3.1 承認待ち

```
申請完了
  ↓
承認者に通知（メール/システム内）
  ↓
承認者が確認
  ├─ 承認 → 申請確定
  └─ 差戻し → 申請者に通知 → 修正して再申請
```

### 3.2 差戻し時の対応

差戻しされた場合の対応は [40_edge_cases.md](./40_edge_cases.md) を参照
